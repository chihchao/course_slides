### 實務上的四個轉化步驟 如果要為一家企業規劃落實方案，可以參考這套流程： #### 1. 關鍵議題鑑定 (Materiality Assessment) 並非所有 SDG 目標都適合每家企業。首先要找出對公司業務「營收影響最大」且「利害關係人最在意」的議題。 > **例子：** 電子製造業會將「電子廢棄物處理」列為優先；而食品零售業則更看重「食物浪費」與「塑膠包裝」。 #### 2. 設定量化基線 (Baselining) 在喊出「減碳 50%」之前，必須先知道現在的數據。 * **行動：** 進行組織碳盤查（ISO 14064）或產品碳足跡計算，這就是 **ESG 中的 E** 的起手式。 #### 3. 制定政策與管理機制 (Governance) SDG 是理想，但沒有制度就無法持久。 * **行動：** 在董事會下設立「永續發展委員會」，將永續表現與高階主管的 KPI 獎金掛鉤。這能確保 **G (公司治理)** 帶動整個行動。 #### 4. 透明揭露與第三方驗證 (Disclosure) 將成果寫入「永續報告書」（如參考 GRI, SASB 指標），並請會計師事務所或認證機構進行簽證。 * **目的：** 讓投資人（ESG 投資者）看到這份成績單，進而提升公司市值。

**講者腳本：** 首先，讓數據說話。這裡有三份權威報告：倫敦經濟政治統計學院關於英國科技業需求的報告、世界經濟論壇關於 2030 年前增長最快職業的報告，以及美國勞工統計局對 2033 年前的趨勢預測。我們會發現，除了 AI、軟體開發這些硬技術職位外，出現了唯一一個帶有「設計」字眼的職業，那就是 UX/UI 設計師。這證明了即便在技術掛帥的時代，設計依然佔據核心地位。

**講者腳本：** 為什麼 UI 設計這麼重要？大家現在手邊應該都有手機或電腦吧。大家有沒有遇過這種情況：有些購物 App 用起來超順手，但換另一個就覺得卡卡的、找不到東西？或者訂機票時，網站讓你很沒安全感？這些「感受」的背後，就是 UI 設計在運作。根據研究，我們每天會打開近 10 個 App，花 3 到 4 小時在手機上。UI 設計的好壞，直接決定了我們這幾小時過得順不順心，這就是它扮演關鍵角色的原因。

**講者腳本：** 那 UX 是什麼呢？全稱是 User Experience，使用者體驗。回到剛剛的餐廳，假設你想吃招牌菜，結果發現菜單最上方已經幫你列好了「當日推薦」還按人氣排序，讓你只花一分鐘就輕鬆點完餐。這個順利、高效、讓人感到滿意的「過程」，就是 UX。UX 的職責是確保產品用起來爽快、沒有障礙，幫助使用者快速達成目標。所以，優秀的體驗一定是 UI（外觀）與 UX（流程）的完美結合。 **UX（User Experience，使用者體驗）** 則是指使用者在使用產品過程中的整體感受與體驗旅程。UX 設計師專注於解決「使用者如何順利完成任務」這個問題，包括從進入產品、搜尋資訊、下單到離開的整個流程。UX 關注的是易用性、流暢度、效率與使用者滿意度。

**講者腳本：** 到底什麼是 UI？UI 的全稱是 User Interface，也就是使用者介面。我們用「餐廳」來比喻：當你走進一家新餐廳，服務生遞給你一台 iPad 點餐。這份電子菜單看起來是很現代還是很傳統？價格看得清楚嗎？按鈕是大是小？顏色是什麼？這些你能直接「看見」的產品樣貌，就是 UI。簡單來說，UI 負責產品看起來的樣子。 **UI（User Interface，使用者介面）** 是指使用者直接接觸到的所有視覺與互動元素。簡單來說，UI 設計師負責產品的「長相」—— 包括按鈕、顏色、字體、排版、圖示（Icon）、整體視覺風格等。UI 的目的是讓產品看起來美觀、易於理解，並能有效傳達品牌訊息。

**講者腳本：** 現在大家有了基本概念，常有人問：UI 設計跟網頁設計有什麼差？嚴格來說，在台灣就業市場這兩者常被視為同一職位，但 UI 設計涵蓋範圍更廣。網頁設計只做網站；但 UI 設計（或稱產品設計）包含了 App、VR 設備、車載系統等任何需要「人機交互」的場景。簡單說，UI 設計服務的是廣泛的數位產品，而網頁設計只專注在網站上。

**講者腳本：** 那跟平面設計的差別呢？一句話概括：平面設計是「靜態」的，像海報、名片，不需要思考使用者怎麼跟它互動。而 UI 設計雖然也重視排版色彩，但它是有重疊的，最關鍵的差異在於「互動」。平面設計師若想轉職 UI，必須轉變思維：不能只做靜態畫面，必須學會思考使用者如何與產品建立長久、持續的互動關係。

**講者腳本：** 最後來到第四部分，UI 設計師到底是怎麼把 App 做出來的？流程主要分三步。第一步是「研究」，我稱之為在黑暗混沌中尋找真相。數位產品通常是為了幫企業獲利，但前提是使用者要用得開心。所以在這階段，我們要用調研、競品分析等方法，同時釐清「使用者真正的需求」以及「企業的商業目標」，找到兩者的交集。

**講者腳本：** 找到目標後，進入第二步「設計」，也就是建構路徑。如果一開始方向錯了，重做成本很高。所以我們會先用線框圖、原型設計做小規模測試，這就是用低成本來「試錯」。確認方向正確後，我們才集中火力設計出那條通往目標的「主路」，確保這條路走起來順暢無阻。

**講者腳本：** 第三步是「交付」。設計圖畫好後，要整理成開發團隊看得懂的格式，讓他們把這條路蓋出來。發布前要修補坑洞（Bug），確保一切流暢。但發布後並不是萬事大吉，我們還需要持續收集反饋、維護產品，這些數據將成為設計下一代更好產品的基礎。

**講者腳本：** 看到這裡，相信你對 UI 設計已經有了完整的認識。下次打開手機 App 時，你可能會用全新的視角去觀察它。每一個好用的產品背後，都有團隊的努力。作為 UI 設計師，看到產品解決了人們的問題、提升了生活品質，那是我們最有成就感的時刻。如果你對這領域感興趣，歡迎訂閱我的頻道。我是 Tony，我們下期影片見！
